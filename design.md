# Headroom — Design Guidelines

UI は **macOS のネイティブメニュー（NSMenu）** で実装する（ADR-0005）。自作の Web UI は使わない。見た目（vibrancy・フォント・ダークモード・選択ハイライト・角丸）はすべて OS 標準に委ね、こちらは**構造と文言とアイコン**だけを定義する。

> 旧版（案A の Web ポップオーバー：カード＋進捗バー）は ADR-0005 で廃止。`src/` の Web 資産と過去の mockup は vestigial。

## 1. 全体の方針

- **OS 標準に委ねる**：色・余白・タイポ・ハイライトは作らない。NSMenu の標準に従う。
- **主役は数字**：各 Usage Window の「残り %」とリセット時刻。
- **クリックできるのは行動だけ**：Tool 見出し（→アプリ起動）・「更新」・「Quit」。情報行（利用枠・エラー）は**無効（グレー）**にする＝ネイティブ標準の情報表示（Codex の "Usage" セクションと同じ）。

## 2. メニュー構造

```
[●] Claude                ← IconMenuItem（クリックで Claude.app 起動）
      5-Hour  ·  残り 77%  ·  2時間54分後     ← 無効（グレー・情報）
      Weekly  ·  残り 89%  ·  6月11日          ← 無効（グレー・情報）
──────────────────────────
更新                       ← 有効（手動取得）
Quit Headroom              ← 有効（終了）
```

- Tool ごとに「見出し（アイコン付き・有効）」＋「利用枠 or 状態行（無効）」を繰り返す。
- 末尾に区切り → 「更新」→「Quit Headroom」。
- メニューバーのトレイアイコンは**テンプレート画像**（モノクロ、ライト/ダーク自動）。

## 3. アイコン（Tool 見出し）

`IconMenuItem` に小さなブランド色アイコンを付ける。当面は生成した円（`brand_dot`）。**後日、各 Tool の実ロゴ PNG（正方形・透過）に差し替える**（`src-tauri/icons/tools/<tool>.png`、`image-png` 機能を有効化）。

| Tool | 暫定ブランド色 (RGB) |
|---|---|
| Claude | `217, 119, 87`（コーラル） |
| Cursor | `90, 90, 95` |
| Codex | `88, 108, 240`（ブルー） |

## 4. 文言フォーマット

- 利用枠行（% 枠）：`    {label}  ·  残り {N}%  ·  {reset}`（先頭空白でインデント）
  - `残り %` = `100 - Consumption` を四捨五入（0 未満は 0 に丸め＝頭打ち）。
  - `reset`：2日以内は相対（`2時間54分後` / `45分後`）、2日以上先は絶対（`6月11日`）。0以下は `まもなくリセット`。
- On-Demand 行（金額枠）：`    On-Demand  ·  ${金額}  ·  {reset}`。% ではなく実額（青天井で上限が無いため。CONTEXT.md の On-Demand）。Cursor で含まれる枠を超えたときのみ表示。
- ラベルは Tool 由来のまま（Claude=`5-Hour`/`Weekly`、Cursor=`Monthly`/`On-Demand`）。

## 5. 状態と文言（取得失敗時は理由＋対処を必ず併記）

成功時は利用枠行を表示。失敗/未接続時は**グレーで「⚠ 理由」＋「対処」の2行**を出す。

**直近値の stale 保持（ガードレール1）**：一度でも取得に成功していれば、その後の取得失敗時も**直近の利用枠行を表示し続け**、末尾に「· 前回値」と理由（⚠）を併記する。429 のときは `retry-after` を尊重し、その間は再取得しない（usage 取得 API の制限であって利用枠とは別物。ハンマリングで制限が永続化するのを防ぐ）。

| 状態 | 理由（1行目） | 対処（2行目） |
|---|---|---|
| 未接続（資格情報なし） | `未接続です` | `Claude にログインすると表示されます` |
| 認証期限切れ（トークン失効） | `認証の有効期限切れ` | `Claude Code を一度使うと自動で更新されます` |
| 資格情報の解析失敗 | `資格情報を解析できません` | `Claude に再ログインしてください` |
| 取得APIの制限 (429) | `取得が一時的に制限中` / `取得が一時制限中（約N分後に再取得）` | `利用枠ではなく取得APIの制限です。後で自動再取得します` |
| 認証エラー | `認証エラー (401/403)` | `Claude を一度起動・使用すると回復します` |
| サーバーエラー | `サーバーエラー (5xx)` | `時間をおいて「更新」してください` |
| 接続不可 | `接続できません` | `ネットワーク接続を確認してください` |
| 応答解析失敗 | `応答を解析できませんでした` | `アプリの更新が必要かもしれません` |

## 6. 更新タイミング（ADR-0005）

起動時に取得 → 以後 **5分間隔**で自動更新（429 回避のため低頻度）→ 「更新」項目で手動取得。NSMenu はメインスレッドでのみ生成・差し替えする。
