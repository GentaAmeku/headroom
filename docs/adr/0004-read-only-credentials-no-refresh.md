# 資格情報は読み取り専用 — 自前リフレッシュも書き戻しもしない

## Context

各 Tool の認証情報（Claude は macOS Keychain `Claude Code-credentials`）は、その Tool 本体（実働の Claude Code 等）と**共有**される。アクセストークンは失効する（Claude は約8時間）。失効時に我々が `/v1/oauth/token` で自前リフレッシュして共有ストアに書き戻すと、リフレッシュトークンのローテーション競合で**本体側の認証を壊し得る**。

## Decision

このアプリは資格情報を**読むだけ**。トークンの自前リフレッシュも、共有ストアへの書き戻しも行わない。トークンが失効していたら取得を諦め、直近の Snapshot を **stale として残しつつ赤系で「更新失敗 ＋ 最終取得時刻」を明示**し、必要なら「その Tool を一度使うと更新されます」等のヒントを出す。トークンは Tool 本体の通常利用によって自然更新される前提。

## Considered Options

- **自前リフレッシュ＋Keychain 書き戻し** — 常に最新を取れるが、共有資格情報への書き込みで本体の認証を壊すリスク。個人/OSS の MVP では割に合わず却下（post-MVP の選択肢としては残す）。

## Consequences

- Tool を日常的に使うユーザーならトークンは常にフレッシュ。長時間使っていない Tool は stale 表示になるが、本体を一度使えば回復する。
- 「共有資格情報ストアにあえて書かない」は意図的な安全境界。将来「stale が不便だからリフレッシュを足そう」とする変更は、この競合リスクを理解した上で行うこと。
