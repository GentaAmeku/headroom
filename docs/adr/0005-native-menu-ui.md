# UI を Web ポップオーバーからネイティブメニュー（NSMenu）に変更する

## Context

当初は案A（Tauri の WebView による自作ポップオーバー、`design.md` 旧版）で UI を作った。vibrancy 等でネイティブに寄せたが、本物の AppKit 部品ではない。ユーザーが「自作より OS 標準パーツに寄せたい（Codex のメニューバーのようなネイティブメニュー）」と要望。Tauri の `Menu` / `MenuItem` / `IconMenuItem` API は**実際の NSMenu を生成**できるため、これを使えば WebView を使わずに OS 標準のメニューを実現できる。

## Decision

メニューバーの UI を **ネイティブメニュー（NSMenu）** で実装する。トレイアイコンのクリックで OS 標準メニューが開き、各 Tool の Usage Window をメニュー項目として並べる。WebView ウィンドウは廃止（`windows: []`）。見た目・挙動（vibrancy・フォント・ダークモード・ハイライト）はすべて OS に委ねる。

- Tool 見出し（アイコン付き `IconMenuItem`）は**クリックで対応アプリを起動**（`open -a <App>`）。
- Usage Window 行と失敗/未接続行は**無効（グレー）の情報表示**。失敗時は「理由＋対処」を併記。
- 表示はテキスト（進捗バーは出さない）。Consumption は「残り X%」で表す。

## Considered Options

- **(B) Web ポップオーバーをネイティブ風に磨く** — 案A＋vibrancy。却下：どれだけ寄せても本物の AppKit にならない。
- **(C) Swift で全面書き換え** — 100% ネイティブだが Tauri（ADR-0002）と動いている実装を捨てるコスト大。ネイティブメニューで目的を達成できるため却下。

## Consequences

- **データ取得方針を更新**：ネイティブメニューは「開く前にデータが用意されている」必要があるため、ADR の「バックグラウンドポーリングなし／開いた時取得」を改め、**起動時＋5分間隔の軽い更新＋手動『更新』**にする（429 回避のため低頻度）。
- **案A の Web 資産は使用しない**（`design.md` 旧版・`docs/design/mockup.html`・`src/` の Web UI は vestigial）。`design.md` はネイティブメニュー前提に書き直す。
- バー等のリッチな視覚表現はできない（テキストのみ）。リッチ表示が必要になったら Web ポップオーバー併用を再検討する。
- メニュー項目アイコンは当面ブランド色の生成アイコン。後日、実ロゴ PNG に差し替える（`image-png` 機能を有効化）。
